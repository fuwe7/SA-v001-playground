# 🏀 Shot Analyzer — анализатор баскетбольного броска

Инструмент для разбора техники броска по видео: распознаёт позу игрока,
считает углы (локоть, колено, стопы), детектирует фазы броска и собирает статистику.

> Проект в активной разработке. Текущий план развития — в [`ROADMAP.md`](ROADMAP.md).
> Сейчас завершён **Этап 0 (навести порядок)**: чистый headless-пайплайн «видео → метрики».

## Как это работает

- **MediaPipe Pose** распознаёт скелет человека на каждом кадре.
- По ключевым точкам считаются углы суставов (`core/detector.py`).
- Стейт-машина определяет фазы броска `IDLE → PREP → RELEASE` и собирает метрики
  по каждому броску (`core/state_machine.py`).
- Единый пайплайн `core/analyzer.py` прогоняет видео и возвращает структурированный результат.

## Установка

Нужен Python 3.10+. GPU не требуется.

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск

```bash
python analyze.py path/to/shot.mp4
# сохранить результат в JSON:
python analyze.py path/to/shot.mp4 --json result.json
```

### Визуализация (видео + скелет + статистика)

```bash
# живые окна: видео со скелетом + окно "Live Stats" (выход — клавиша q)
python analyze.py path/to/shot.mp4 --show

# то же самое, но записать в файл (работает и без дисплея):
python analyze.py path/to/shot.mp4 --save-video annotated.mp4
```

Пример вывода:

```
Кадров всего: 240
Кадров с распознанной позой: 232
Найдено бросков: 3

#  | elbow | knee_load | feet_diff | grade
--------------------------------------------------
1  | 168   | 132       | 8         | Great Arc
2  | 151   | 121       | 12        | OK
3  | 142   | 158       | 9         | Short Arm (No Legs)
```

Для каждого засчитанного броска дополнительно считается набор **3D-углов суставов**
(обе руки, плечи, бёдра, оба колена, наклон корпуса) по world landmarks MediaPipe —
они устойчивее к ракурсу. Углы суставов, которые в данном ракурсе плохо видно,
помечаются `н/д` (в JSON — `null`). Виден в выводе CLI и в `--json`.

Программа также **сама определяет ракурс** съёмки (`side` / `front/back` / `angled`)
и подсказывает, каким метрикам в нём можно доверять (углы локтя/колена точнее всего
видно сбоку). Ракурс показывается в CLI, в `--json` и в окне визуализации.

Лучшие результаты — если снимать **сбоку** (виден угол локтя и колена).

## Структура

```
core/
  detector.py        # PoseDetector — обёртка над MediaPipe Pose
  state_machine.py   # ShotDetector — фазы броска и метрики
  analyzer.py        # analyze_video() — единый пайплайн (источник правды)
  biomechanics.py    # 3D-углы суставов по world landmarks (устойчивы к ракурсу)
  view.py            # авто-определение ракурса камеры (side / front-back / angled)
  visualizer.py      # видео + скелет + панель статистики (--show / --save-video)
  ball_tracker.py    # экспериментально, пока НЕ подключено (трекинг мяча, Этап 5)
analyze.py           # CLI-вход
tests/               # smoke-тесты
legacy/              # старые прототипы (десктоп main.py, streamlit app.py) — не поддерживаются
ROADMAP.md           # план развития проекта
```
